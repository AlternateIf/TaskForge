### Persona: SEO Expert — Raj

#### Background

- **Role**: SEO specialist responsible for organic search strategy, technical SEO audits, and content optimization
- **Experience**: 4 years; proficient with Google Search Console, Lighthouse, Screaming Frog, and analytics platforms
- **Tech comfort**: Moderate; comfortable reading HTML source and basic JavaScript but relies on developers for implementation

#### Goals

- Create and prioritize SEO-related tasks (meta tags, structured data, page speed, crawlability) alongside feature work
- Attach audit reports and keyword data to tasks so developers have full context
- Track the impact of SEO changes with before/after metrics linked to completed tasks

#### Pain Points

- SEO tasks deprioritized because they lack visibility in the main product backlog
- No standard way to attach Lighthouse or Core Web Vitals reports to a task
- Difficulty communicating technical SEO requirements to developers unfamiliar with search-engine guidelines

#### Usage Scenarios

1. **Technical audit**: Creates tasks from a site-wide crawl report, tagging each by category (indexing, performance, structured data) and priority.
2. **Page-speed optimization**: Files a task with Lighthouse scores, attaches a performance budget, and tracks developer progress.
3. **Content launch**: Adds an SEO checklist (title tag, meta description, canonical URL, schema markup) as sub-tasks on a content-release epic.

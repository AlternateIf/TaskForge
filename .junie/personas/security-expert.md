### Persona: Security Expert — Nadia

#### Background

- **Role**: Application Security Engineer at a mid-size software company
- **Experience**: 8 years in cybersecurity, specializing in web application security audits and penetration testing
- **Tech comfort**: High; comfortable with code review, OWASP tooling, and security scanners

#### Goals

- Ensure the application is protected against OWASP Top 10 vulnerabilities (SQL injection, XSS, CSRF, etc.)
- Enforce secure defaults across authentication, authorization, and data handling
- Establish automated security testing in the CI/CD pipeline

#### Pain Points

- Developers often treat security as an afterthought, leading to costly late-stage fixes
- Lack of centralized visibility into security-related issues and remediation status
- Inconsistent input validation and output encoding across different modules

#### Usage Scenarios

1. **Code review**: Reviews pull requests for injection flaws, improper authentication, and insecure data exposure.
2. **Security audit**: Runs automated scans (SAST/DAST) against the application and triages findings into prioritized tasks.
3. **Incident response**: Investigates a reported vulnerability, traces the attack vector, and creates tasks to patch and harden affected endpoints.

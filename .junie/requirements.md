# TaskForge — Requirements Router

Use this router to avoid loading the full requirements corpus when it is not necessary.

## Read Order
1. `requirements-mvp.md` for current implementation and feature delivery.
2. `requirements-future.md` for planning beyond MVP (Phase 2/3).

## When to Load Which File
- MVP implementation, bugfixes, acceptance checks: `requirements-mvp.md`
- Sprint planning and roadmap shaping: `requirements-mvp.md` + `requirements-future.md`
- Broad product strategy discussions and complete traceability audits: `requirements-mvp.md` + `requirements-future.md`

## Source of Truth
- `requirements-mvp.md` and `requirements-future.md` are the source of truth.

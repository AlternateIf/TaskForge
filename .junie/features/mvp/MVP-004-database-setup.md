# MVP-004: Database Schema & Connection

## Description
Set up Drizzle ORM with MariaDB connection, define MVP database schema (users, sessions, organizations, roles, permissions, projects, workflows, labels, tasks, checklists, dependencies, comments, activity log, notifications, attachments), create initial migration, and seed script.

## Personas
- **Marcus (Backend)**: Type-safe database access from day one
- **Dana (Systems)**: Connection pooling and timeout configuration

## Dependencies
- MVP-001 (packages/db exists)
- MVP-002 (MariaDB container running)

### Schema details (MVP entities only)

Reference `data-model.md` for full attribute lists. MVP includes:
- **User**: id (UUID), email, password_hash, display_name, avatar_url, mfa_enabled, mfa_secret, email_verified_at, last_login_at, created_at, updated_at, deleted_at
- **OAuthAccount**: id, user_id, provider, provider_user_id, access_token, refresh_token, expires_at
- **Session**: id, user_id, token_hash, ip_address, user_agent, expires_at, created_at
- **Organization**: id (UUID), name, slug, logo_url, settings (JSON), trial_expires_at, created_at, updated_at, deleted_at
- **OrganizationMember**: id, organization_id, user_id, role_id, joined_at
- **Role**: id, organization_id (nullable), name, description, is_system
- **Permission**: id, role_id, resource, action, scope
- **Project**: id (UUID), organization_id, name, description, slug, color, icon, status, created_by, created_at, updated_at, deleted_at
- **ProjectMember**: id, project_id, user_id, role_id
- **Workflow**: id, project_id, name, is_default
- **WorkflowStatus**: id, workflow_id, name, color, position, is_initial, is_final
- **Label**: id, project_id, name, color
- **Task**: id (UUID), project_id, title, description, status_id, priority, assignee_id, reporter_id, parent_task_id, due_date, start_date, estimated_hours, position, created_at, updated_at, deleted_at
- **TaskDependency**: id, task_id, depends_on_task_id, type
- **TaskLabel**: task_id, label_id
- **TaskWatcher**: task_id, user_id
- **Checklist**: id, task_id, title, position
- **ChecklistItem**: id, checklist_id, title, is_completed, position, completed_by, completed_at
- **Comment**: id, entity_type, entity_id, author_id, body, parent_comment_id, created_at, updated_at, deleted_at
- **ActivityLog**: id, organization_id, actor_id, actor_display, entity_type, entity_id, action, changes (JSON), created_at
- **Notification**: id, user_id, type, title, body, entity_type, entity_id, read_at, created_at
- **NotificationPreference**: id, user_id, event_type, channel, enabled
- **Attachment**: id, entity_type, entity_id, uploaded_by, filename, mime_type, size_bytes, storage_path, version, scan_status, created_at

### Connection configuration
```typescript
// client.ts
import { drizzle } from 'drizzle-orm/mysql2';
import mysql from 'mysql2/promise';

const pool = mysql.createPool({
  uri: process.env.DATABASE_URL,
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0,
  connectTimeout: 5000,
});
```

### Seed script
- Creates 1 organization with default roles (Super Admin, Admin, PM, Team Member, Guest)
- Creates 3 test users (admin, pm, member) with known passwords
- Creates 1 project with default workflow (To Do, In Progress, Review, Done)
- Creates 5 sample labels
- Creates 10 sample tasks across different statuses

## Acceptance Criteria
- [ ] `pnpm --filter db migrate` creates all MVP tables in MariaDB
- [ ] `pnpm --filter db seed` populates test data
- [ ] `pnpm --filter db studio` opens Drizzle Studio for visual DB browsing
- [ ] All foreign keys and indexes are correctly defined
- [ ] UUID primary keys on main entities (User, Organization, Project, Task)
- [ ] Soft delete (deleted_at) on User, Organization, Project, Task, Comment
- [ ] ActivityLog is append-only (no update/delete in schema)
- [ ] Factory functions generate valid test data for all entities
- [ ] Connection pool settings are configurable via environment variables

## 1. Drag-and-Drop Library

**Priya:** `@dnd-kit/core` + `@dnd-kit/sortable` are not yet in `package.json`. These are the
libraries specified in the feature. We'll install `@dnd-kit/core`, `@dnd-kit/sortable`, and
`@dnd-kit/utilities`. All three are required for sortable drag-and-drop with accessible keyboard
support.

**Ava:** Make sure the DragOverlay animates card movement. The card should "lift" slightly (subtle
scale + shadow) when picked up. Drop indicator should be the 2px brand-primary line, not a ghost
card.

**Derek:** I need keyboard drag-and-drop to work — Space to pick up, arrow keys to move between
columns and positions, Space to drop. Screen reader announcements on move.

**Priya:** `@dnd-kit` has first-class keyboard support via the `KeyboardSensor`. I'll configure
custom keyboard coordinates for column-to-column movement.

**Decision:** Install `@dnd-kit/core`, `@dnd-kit/sortable`, `@dnd-kit/utilities`. Use
`DragOverlay` for the lifted card. Drop indicator = `2px solid brand-primary` line component.
Keyboard sensor configured alongside pointer sensor.

## 2. Move Task API Endpoint

**Priya:** The spec says `PATCH /tasks/:id/position`. Our api-conventions use `PATCH /resource/:id`
for partial updates. Moving a task updates `statusId` and `position` — both are task fields, so
`PATCH /tasks/:id` with `{ statusId, position }` follows the conventions and is equivalent.

**Sarah:** Either works from my perspective — just needs to be optimistic so the card moves
immediately without waiting for the server.

**Decision:** Use `PATCH /tasks/:id` with `{ statusId, position }` for task movement (follows
api-conventions). Optimistic update: update React Query cache immediately, revert on API error
with a toast notification.

## 3. Project List Card Shape

**Priya:** The API will return projects with at minimum: `id`, `name`, `color`, `description`,
`icon`, `taskCount`, `memberCount`, and `updatedAt`. We'll define a `Project` type in
`api/projects.ts` and a `ProjectMember` type for member avatars.

**Ava:** Project cards in the grid: colored accent bar or dot using `project.color`, project name
as the card title, member avatar stack (max 3 + overflow count), task count badge. Cards use
`rounded-radius-lg` (8px) with `shadow-1` and `hover:shadow-2` transition. Background is
`bg-surface-container-lowest`.

**Sarah:** The "New Project" button should be a primary gradient button, consistent with the
sidebar "New Project" CTA.

**Decision:** Project card shows: color dot, name, description (truncated), member avatar stack
(max 3 + "+N"), task count. Grid: 1 col mobile / 2 col md / 3 col lg. Create button is primary
gradient variant.

## 4. Kanban Column Quick Add

**Priya:** The "quick add" at column bottom creates a task with title only. It sends
`POST /projects/:projectId/tasks` with `{ title, statusId }`. The task inherits the column's
status.

**Sarah:** Must feel instant — no dialog, just a simple input at the bottom of the column that
appears when I click "Add task" or the FAB. After submit the card appears immediately (optimistic).

**Ava:** Quick add input: subtle `bg-surface-container/50` background, `border border-border`
dashed style, placeholder "Add task..." Submit on Enter key, dismiss on Escape. No submit button
visible — the Enter affordance is enough once focused.

**Derek:** I also need a "create task" FAB on mobile (above bottom nav). On desktop, the
"+ Add task" text at the column bottom suffices.

**Decision:** Quick add = inline input at column bottom, triggered by click on "Add task" row or
FAB (mobile). Desktop: shows text trigger; click reveals input. Mobile: FAB opens the create-task
dialog (full form). Enter submits, Escape cancels. Optimistic — card appears in column immediately.

## 5. List View Pagination

**Marcus (via prior meeting decisions):** Cursor-based pagination only. No page numbers.

**Priya:** I'll use `useInfiniteQuery` from TanStack Query with `?cursor=&limit=50`. The table
renders all loaded rows; a "Load more" button appears when `hasMore: true`. Alternatively, an
IntersectionObserver on the last row triggers auto-load for infinite scroll. I'll do the button
first (simpler to test), then add the intersection observer.

**Derek:** I need to be able to select all loaded rows for bulk actions, not just the current
"page". With infinite scroll and a "Select all" checkbox in the header, it selects all loaded
tasks. Fine for MVP.

**Decision:** `useInfiniteQuery` with cursor-based API. First render: 50 tasks. "Load more" button
when `hasMore: true`. Select-all applies to all currently loaded rows. Intersection observer for
auto-load can be added later without API changes.

## 11. Mobile Kanban: Scroll Snap

**Priya:** CSS `scroll-snap-type: x mandatory` on the board container. Each column: `scroll-snap-
align: start`. Works without JS. On mobile, users swipe between columns. Column width: 85vw so
the next column peeks.

**Lena (via prior meeting):** Horizontal swipe on mobile is a well-established pattern (Trello,
Linear). The scroll snap behavior provides tactile feedback.

**Decision:** Mobile board: `overflow-x-auto scroll-smooth` + `snap-x snap-mandatory` Tailwind
classes. Each column: `snap-start min-w-[85vw]`. No dots/pagination indicators for MVP.
